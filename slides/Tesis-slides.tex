\frame{\frametitle{Sistema propuesto}
\vfill
\textbf{Basado en:}
\begin{description}
 \item[Time-hopping Spread Spectrum:] El tiempo de transmisión se selecciona mediante un algoritmo generador de números pseudoaleatorios (\alert{PRBS}).
 \item[Filtro de Bloom:] Provee \alert{corrección de errores} asimétrica (en un canal Z)\vfill
 \item[Minimización de peso de Hamming:] \alert{reducción} de símbolos problemáticos en el canal Z\vfill
\end{description}
\vfill
\pause
\textbf{Ventajas:}
  \begin{itemize}
  \item Punto-a-punto y Punto-a-\alert{multipunto}\vfill
  \item \alert{Privacidad}\vfill
  \end{itemize}
\vfill

\subsection{Minimización del peso de Hamming}

\frame{\frametitle{Minimización del peso de Hamming: origen}
\framesubtitle{Origen de la técnica}
  \begin{itemize}
  \item Peso de Hamming: cantidad de '1's en un dígito binario.
  \item Ej.: $HW(00101010) = 3$
  \item En un canal Z, los '1' producen interferencia, los '0', no.
  \item Es un \alert{sistema de numeración}.
  \end{itemize}
}

\frame{\frametitle{Minimización del peso de Hamming: origen}
\framesubtitle{Sistemas de numeración}
  \begin{itemize}
  \item Mas extendido: números arábigos.
  \item Originalmente de India, documentado e introducido a Europa por \alert{Al-Jwarizmi}.
  \end{itemize}
Dada un conjunto de $n$ dígitos ordenados $d$,

con posición $p$ y base $B$, el valor numérico $VN$ es:

$$VN=\sum_{n=1}^{n} B^{p}*d$$
Ej.
$$1685=10^{3}*1+10^{2}*6+10^{1}*8+10^{0}*9$$

}

\frame{\frametitle{Minimización del peso de Hamming: origen}
\framesubtitle{Sistemas de numeración: Ejemplos}
\begin{columns}
  \begin{column}{0.60\textwidth}

  \begin{itemize}
  \item Bases no estandard, Ej.: Negabinario, base -2
        $$VN=\sum_{n=1}^{n} (-2)^{p}*d$$
  \item Sistema numeral \textit{Quater-imaginario}, base $-2i$, por \cite{knuth1960imaginary}.
        $$VN=\sum_{n=1}^{n} (2i)^{p}*d$$
  \end{itemize}

  \end{column}
  \begin{column}{0.40\textwidth}
  \begin{figure}[t]
  \centering
    \includegraphics[width=1.5in]{graphs/twindragon.png}
     
    Dragón de Davis-Knuth : fractal asociado a la base $(-1 \pm i)$
\end{figure}


  \end{column}
\end{columns}

  
}

\frame{\frametitle{Minimización del peso de Hamming: origen}
\framesubtitle{Sistemas de numeración: Ejemplos}
\begin{columns}
  \begin{column}{0.60\textwidth}

  \begin{itemize}
  \item Códigos no regulares: Código Gray
  \item \alert{Dígitos con signo}. Ej.:
    $$1685=1000*1+100*6+80*10+5$$
    pero también:
    $$1685=169(-5)=1000*1+100*6+90*10+(-5)$$
    y también:
    $$1685=169\bar{5}=17\bar{1}\bar{5}$$
  \item Usado en el lenguaje Punjabi.
    
  \end{itemize}

  \end{column}
  \begin{column}{0.40\textwidth}
  \begin{figure}[t]
  \centering
    \includegraphics[width=1.8in]{graphs/gray.png}
     
    Código Gray
\end{figure}


  \end{column}
\end{columns}
  
}

\frame{\frametitle{Minimización del peso de Hamming: origen}
\framesubtitle{Sistemas de numeración: Dígitos signados}
\begin{columns}
  \begin{column}{0.60\textwidth}

  \begin{itemize}
  \item Existe más de una representación posible por número.
  \item La representación canónica de dígitos signados (CSD) es util:
  
        La probabilidad digitos no-cero es $\frac{1}{3}$.
        
        Usando dígitos binarios es $\frac{1}{2}$.
        
  \item Primeramente el sistema propuesto utilizó esta representación.
  \item Luego se utilizó una tabla con símbolos optimizados con un peso de Hamming fijo.
  \end{itemize}

  \end{column}
  \begin{column}{0.40\textwidth}
    Binario: \tab{$001010111$}
    
    CSD:     \tab{$010\bar{1}0\bar{1}00\bar{1}$}
  \end{column}
\end{columns}
  
}

\begin{frame}[fragile]{Simulación: software de prueba}

Ejemplo de linea de comando para lanzar la simulación:

\small
\begin{verbatim}
 ./rsenc <testfile.in | ./scrambler ${SCRAMBLEBLOCK} >rs.out
 ./bfenc ${CLIENTES} < rs.out | ./noisesim -c ${CLIENTES} -r 16.6 >bfenc.out
 ./bfdec ${CLIENTES} <bfenc.out >bf.out
 ./descramble ${SCRAMBLEBLOCK} <bf.out | ./rsdec >rsdec.out
\end{verbatim}
\normalsize

El BER (\textit{bit error rate}) se calcula con la diferencia entre \textbf{testfile.in} y \textbf{rsdec.out}.
